Implémenter une structure HTML5 pertinente

  • <header>
  • <nav>
  • <main>
  • <aside>
  • <footer>
  • <section>
  • <article>

Implémenter les éléments de navigation interne

  • Liens d'accès rapide
  • role="banner"
  • role="search"
  • role="navigation"
  • role="main"
  • role="complementary"
  • role="contentinfo"

Avec NVDA :

  • naviguer entre les régions : D et Shift D
  • afficher la liste des régions : inser F7

Exercice 2 : le contenu

Réparer les éléments du contenu

  • Titrage
  • Listes
  • Images
  • Tableau

Avec NVDA :

  • naviguer entre les titres : H et Shift H
  • afficher la liste des titres : inser F7
  • naviguer entre les listes : L et Shift L
  • naviguer entre les images : G et Shift G
  • naviguer entre les tableaux : T et Shift T
    • naviguer entre les lignes d'un tableau : Ctrl Alt Haut et Ctrl Alt Haut
    • naviguer entre les cellules d'une ligne : Haut et Bas

Répérer les erreurs et les corriger.

Vous devez vérifier que le contenu et ses fonctionnalités restent compréhensibles sans la mise en forme et que les utilisateurs de la navigation au clavier peuvent se situer dans la page.

Exercice 4: présentation, agrandissement des tailles de caractères.

Repérer les erreurs qui apparaissent à l'agrandissement de la page et linéariser le contenu.

Des problèmes apparaissent à l'agrandissement des caractères, repérez les et réparez les.

Le contenu doit rester lisible (texte, contrastes etc.) quelque soit le zoom texte.

Vous avez une mise en page dont les blocs sont constitués de 2 colonnes. À l'agrandissement du texte (zoom texte seul), le contenu doit se linéariser pour n'être présenté que sur une seule colonne.

Aidez-vous des media-queries et des points de ruptures en em. Pour rappel, une media-queries se déclare : @media screen and (max-width:xem){.monselecteur{}}. Les points de rupture sont laissés à votre appréciation.

Exercice 6: JavaScript et ARIA.

Toogle Button

Le bouton "comandez une machine" gère une zone affichée/masquée.

Vous devez rendre accessibles ce bouton et l'événement qui est géré.

Dans le fichier script.js, le bloc de code qui contrôle cette zone est situé entre les lignes 3 et 21.

Ces propriétés sont à utiliser, au moins :

  • aria-controls
  • aria-hidden
  • aria-expanded

Fenêtre modale

Le bouton "Qui est Alan Turing ?" gère une fenêtre modale.

Vous devez vérifier que le motif de conception de la fenêtre modale est correct et réparer les erreurs.

Dans le fichier script.js, le bloc de code qui contrôle la fenêtre modale débute à la ligne 241.

Le motif de conception fenêtre modale est consultable à cette addresse : Dialog modal

Contrôle de saisie de formulaire

Lorsque le formulaire est envoyé et contient des erreurs, le contrôle de saisie affiche des messages d'erreur.

Vous devez implémenter une méthode de votre choix pour garantir que ces messages d'erreurs seront correctement restitués à la prise de focus sur les champs en erreur.

Vous devez vérifier que les messages d'erreurs sont corrects.

Dans le fichier script.js, le bloc de contrôle de saisie est situé à la ligne 37.

Panier d'achat

L'ajout d'une couleur de machine, met à jour le panier d'achat, visuellement positionné en haut à droite.

Vous devez implémenter une méthode de votre choix pour garantir aux utilisateurs l'accès à l'information.

Dans le fichier script.js, le bloc de gestion du panier est situé à la ligne 126.

Ajout de fichiers

Le formulaire dispose d'un champ de téléversement de fichiers. Vous pouvez téléverser plusieurs fichiers, et vous pouvez également les supprimer individuellement.

La problématique ici est la gestion du focus pour les utilisateurs claviers : lorsqu'un fichier est ajouté ou supprimé, où doit être repositionné le focus ?

Dans le fichier script.js, le bloc de gestion du téléversement des fichiers est situé à la ligne 188.
